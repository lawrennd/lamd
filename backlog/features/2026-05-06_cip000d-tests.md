# Task: Write tests for CIP-000D SVG manim pipeline

## Description

Write the test suite for the `--to manim-svg` pipeline in `lamd/tests/test_mdpp_manim_svg.py` and extend `lamd/tests/test_svg_to_html.py` (or create it). This is Phase 5 of CIP-000D. The pattern follows `test_mdpp_manim.py` (CIP-000C).

## Acceptance Criteria

### Macro expansion tests (`test_mdpp_manim_svg.py`)

- [ ] `\newslide{Title}` in `manim-svg` format â†’ contains `self.play(FadeIn(lamd_text` and does **not** contain `self.next_slide()`
- [ ] `\slides{text}` in `manim-svg` format â†’ contains `self.play(FadeIn(lamd_text(r"""text""")))`
- [ ] `\slidesmanim{raw_code}` in `manim-svg` format â†’ `raw_code` appears verbatim in output
- [ ] `\slidesmanim{raw_code}` in `html` format â†’ empty string (no-op; regression check)
- [ ] `\notes{text}` in `manim-svg` format â†’ empty string (no-op)
- [ ] `\section{Title}` in `manim-svg` format â†’ contains `self.play(FadeIn(lamd_text` and does **not** contain `self.next_slide()`

### Integration test

- [ ] `mdpp --to manim-svg` on a minimal fixture talk (two `\newslide` + one `\slides`) â†’ output file passes `python -m py_compile`
- [ ] Output header contains `config.renderer = RendererType.SVG` and `class Talk(Scene):`
- [ ] Output does **not** contain `self.next_slide()` anywhere

### HTML generator tests (`test_svg_to_html.py`)

- [ ] Given a mock animation root with `animation_0/animation.json`, `animation_1/animation.json`, and `animation_2/animation.json`, the output HTML contains exactly 3 `<section data-manim-svg=` elements
- [ ] The `data-manim-svg` attribute values are correctly relative to the output HTML file location
- [ ] `--title "My Talk"` produces a leading title `<section>` before the animation sections
- [ ] Output HTML contains `manim-svg` plugin reference (either inline or as a `<script src=...>`)

## Implementation Notes

- Place tests in `lamd/tests/test_mdpp_manim_svg.py` and `lamd/tests/test_svg_to_html.py`
- Use `pytest` and `tmp_path` fixture for file-based tests
- Mock animation directories can be created with `pathlib.Path.mkdir()` + `json.dump()` â€” no actual SVG files needed for the HTML generator tests
- The `py_compile` integration test requires `mdpp.py` to be importable; follow the pattern in `test_mdpp_manim.py`
